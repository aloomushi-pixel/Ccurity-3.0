1: Seleccionamos al cliente o al colaborador
2: Subimos el contenido del contrato
3: Previsualizar el contrato
4: Enviar para firma
5: Se generaran dos tokens: uno para cliente, el otro para proveedor, cuando es un contrato con colaborador entonces nosotros somos el cliente y el colaborador se vuelve el proveedor
6: El contrato se publicara con una URL jerarquizada y estandarizada con fecha y hora de generación. 
7: En la parte inferior del contrato habrá un histórico que mostrara la IP, del cliente que consulto la url y su token definira su rol en el histórico. y su accion. visualizar, firmar o comentar
8: El comentario se guardara en un control de versiones mismo para el contrato. cuando el admin modifique el contrato, se actualizara el control de versiones tipo GitHub y este podrá consultarse también:
9: Al firmar habrá las siguientes utilidades:
a- una casilla doble de verificación donde los firmantes aceptaran dos veces una casilla para aceptar que quiere usar medios digitales para la firma del contrato y la otra para aceptar que ya leyó todo el contenido y esta completamente de acuerdo con el contrato en el orden que sea mas eficiente
b- se abrirá la cámara (solo cámara, no se permite subir medios) para tomar una selfie del firmante
c- se abrirá la cámara para tomar foto del frente del ine
d- se abrirá cámara para tomar foto del reverso del ine
e- siguiente se abrirá un modal para firmar con el dedo, una firma autógrafa con el dedo
f- se publicara en la misma url el contrato generado con todo y sus medios subidos y se noitificara a la otra parte que ya se firmo el contrato
g- una vez firmado el contrato por ambas partes se actualizara la url con todos los contenidos actualizados 
h- Se enviaran copias a los firmantes por email con su contrato en .HTML y la url para consultarlo en linea.

10: Automatizaciones: 
a- cuando se genera un contrato se notifica a ambas partes
b- cuando se comenta se notificara a ambas partes
c- cuando se modifica se notificara a ambas partes
d- cuando se firma se notificara a ambas partes
e- cuando ambos firmaron se notificara a ambas partes
f- En la plantilla de HTML se mostrara todo el contrato, su control de versiones con comentarios, sus firmas, fotos el histórico todo. y se mostrara un mensaje: porfavor no elimine este email, el contrato tiene una vigencia de 6 meses dentro de nuestros servidores.
